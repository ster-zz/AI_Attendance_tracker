📂 2. Database & Data Persistance (database.py)

Using SQLite3 for robust, serverless data management.

🔹 2.1 Schema Initialization

def init_db():
    conn = get_db_connection()
    cursor = conn.cursor()

    # Table: Students - Stores names and face encodings
    cursor.execute('''
        CREATE TABLE IF NOT EXISTS Students (
            id INTEGER PRIMARY KEY AUTOINCREMENT,
            name TEXT NOT NULL,
            face_encoding BLOB
        )
    ''')

    # Table: Attendance - Logs every identification event
    cursor.execute('''
        CREATE TABLE IF NOT EXISTS Attendance (
            id INTEGER PRIMARY KEY AUTOINCREMENT,
            session_id INTEGER NOT NULL,
            student_id INTEGER NOT NULL,
            status TEXT NOT NULL,
            timestamp TEXT NOT NULL,
            FOREIGN KEY(student_id) REFERENCES Students(id)
        )
    ''')
    conn.commit()
    conn.close()
Plain English Explanation: This code builds the system's "Memory." We use SQL (Structured Query Language) to define tables. The Students table stores who the students are, and Attendance stores when they appeared. We use FOREIGN KEY to link these tables, ensuring that we never lose track of which student belongs to which record.

🔹 2.2 Automated Student Synchronization

def sync_students_from_images():
    images_dir = os.path.join(BASE_DIR, 'data', 'student_images')
    conn = get_db_connection()
    cursor = conn.cursor()
    
    # Identify existing students
    cursor.execute('SELECT name FROM Students')
    existing_students = set(row['name'] for row in cursor.fetchall())
    
    # Auto-add names based on folder names
    for item in os.listdir(images_dir):
        if os.path.isdir(os.path.join(images_dir, item)):
            if item not in existing_students:
                cursor.execute('INSERT INTO Students (name) VALUES (?)', (item,))
    conn.commit()
Plain English Explanation: This is a convenience feature for teachers. Instead of manually entering student names into a form, the teacher just creates a folder named (e.g., "Paul") and puts his photo inside. This script automatically detects new folders and adds them to the database, saving time and preventing typing errors.

📂 5. Compliance & ID Logic (id_detection_module.py)

Advanced heuristics checking for physical ID cards and school lanyards.

🔹 5.1 Color Space Masking (HSV)

# Convert to HSV for robust color recognition in varied lighting
hsv = cv2.cvtColor(chest_roi, cv2.COLOR_BGR2HSV)

# Define the "Orange" mask for school lanyards
lower_orange = np.array([10, 100, 100])
upper_orange = np.array([25, 255, 255])
mask = cv2.inRange(hsv, lower_orange, upper_orange)
Plain English Explanation: Classrooms have different lighting throughout the day. Standard colors (RGB) shift too much in the sun. We convert the image to **HSV** (Hue, Saturation, Value), which isolates the "Hue" (color). This makes the system reliably see the orange lanyard even if the sun is shining directly on it.

🔹 5.2 Geometric Validation

# Calculate the shape (Aspect Ratio) of detected objects
w, h = rect[1] # width and height
aspect_ratio = w / float(h)

# Check if the object is shaped like a standard ID card
if (1.2 <= aspect_ratio <= 1.7) or (0.45 <= aspect_ratio <= 0.85):
    return True # ID Confirmed!
Plain English Explanation: To prevent the system from getting fooled by an orange shirt, we add a "Shape Check." We measure the width and height of any orange object. Since ID cards are standard rectangles (about 1.5x wider than tall), we check the **Aspect Ratio**. If the math matches a rectangle, the AI confirms the ID is present.
